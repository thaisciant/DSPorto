01 / 16
O Problema

Por que precisamos de tokens?

🔀

Inconsistência visual

Sem tokens, cada componente usa valores hard-coded diferentes. Uma mesma "cor primária" pode ter 5 variações no app.

⏳

Retrabalho constante

Mudar uma cor ou espaçamento exige atualizar dezenas de arquivos manualmente. Uma alteração vira dias de trabalho.

🗣️

Falha de comunicação

Design fala "azul primário", dev implementa "#0047CC" e depois "#003087". Não há linguagem comum entre as áreas.

🌙

Temas impossíveis

Dark mode, alto contraste, white-label? Sem tokens semânticos, cada novo tema exige uma refatoração completa.

📏

Escala não padronizada

Espaçamentos e tamanhos arbitrários comprometem o ritmo visual e dificultam a criação de novos componentes alinhados.

🔧

Dev sem especificação

Componentes do Figma não estão prontos para desenvolvimento. Os valores precisam ser extraídos manualmente a cada entrega.

Arquitetura

5 Coleções · Do Valor Bruto ao Produto Final

Cada coleção tem uma responsabilidade única. Um token nunca pula camadas — sempre referencia a coleção imediatamente anterior.

Coleção 01
Primitive
Todos os valores brutos e absolutos do sistema. Nenhuma semântica — apenas a paleta de possibilidades.
color.blue.500 typography.size.16 numeric-values.8 breakpoint.md aspect-ratio.16-9
→
Coleção 02
Brand
Identidade da marca Porto Seguro. Mapeia primitivos para intenções de marca — cor, tipo e forma.
color.brand.primary typography.heading.lg numeric-values.radius.sm breakpoint.sm aspect-ratio.card
→
Coleção 03
Theme
Variações de contexto visual: light, dark, alto contraste. Altera a aparência sem mudar os componentes.
color.surface.default color.text.primary typography.body.md breakpoint.lg aspect-ratio.hero
→
Coleção 04
Channel
Customizações por produto ou canal — SuperApp, Portal Web, Parceiros. Sobrescreve apenas o necessário.
color.surface.default color.accent.primary typography.brand.heading breakpoint.channel aspect-ratio.banner
→
Coleção 05
Platform
Saída técnica para cada plataforma. Transforma tokens em CSS vars, Swift enums, Kotlin objects, SCSS vars.
--color-surface-default DSTokens.Color.surface $color-surface-default @color/surface_default tokens.color.surface
Grupos em todas as coleções → color typography numeric-values breakpoint aspect-ratio

Processo

Fluxo de Trabalho

Uma etapa por coleção. O trabalho avança do mais bruto ao mais específico — cada entregável alimenta a próxima etapa.

Etapa 01

Coleção Primitive

Auditar o Figma e criar a escala base: paleta de cores, tipografia, numeric-values, breakpoints e aspect-ratios.

↳ primitive.json
Etapa 02

Coleção Brand

Mapear primitivos para a identidade Porto Seguro: cor de marca, tipografia e forma. Referencia apenas Primitive.

↳ brand.json
Etapa 03

Coleção Theme

Definir variações visuais: light, dark, alto contraste. Tokens de surface, text e border referenciando Brand.

↳ theme.light.json · theme.dark.json
Etapa 04

Coleção Channel

Customizações por produto: SuperApp, Portal, Parceiros. Sobrescreve somente o que difere do Theme base.

↳ channel.superapp.json
Etapa 05

Coleção Platform

Style Dictionary transforma o JSON em artefatos: CSS vars, TS object, Swift enum, Kotlin object, SCSS vars.

↳ platform.css · .ts · .swift · .kt

Saída técnica

Entrega Pronta para Desenvolvimento

O JSON das 5 coleções alimenta o Style Dictionary, que gera artefatos de plataforma automaticamente. Zero trabalho manual para os devs.

🗂️ JSON · Estrutura das Coleções

// brand.json
{
  "color": {
    "primary": { "value": "{primitive.color.blue.500}" },
    "secondary": { "value": "{primitive.color.orange.500}" }
  },
  "typography": {
    "heading": { "lg": { "value": "{primitive.typography.size.32}" } }
  },
  "numeric-values": {
    "radius": { "sm": { "value": "{primitive.numeric-values.8}" } }
  }
}
// theme.dark.json sobrescreve apenas:
{ "color": { "surface": { "default": {
    "value": "{primitive.color.gray.900}" } } } }

08 / 16
Insumos necessários

O que precisamos para começar

Para criar os tokens, precisamos dos valores que já existem no produto. Quatro insumos desbloqueiam todo o trabalho.

01

Paleta de Cores

Os valores hexadecimais das cores usadas no SuperApp.

  • Cores da marca PS
  • Cores de feedback (erro, sucesso, alerta)
  • Escala de neutros
  • Gradientes (se houver)
02

Escala Tipográfica

Fontes, tamanhos, pesos e alturas de linha em uso.

  • Fonte principal (nome)
  • Tamanhos (xs, sm, md, lg…)
  • Pesos usados
  • Line-height por nível
03

Espaçamento

A base da grade de espaçamento usada nos layouts.

  • Base unit (4px? 8px?)
  • Escala de tamanhos
  • Padding interno dos componentes
  • Gaps entre elementos
04

Acesso ao Figma

Acesso ao arquivo do UI Kit para auditoria completa.

  • Link do arquivo Figma
  • Permissão de visualização
  • Lista de componentes existentes
  • Responsável pelo arquivo
09 / 16
Cronograma

Próximos Passos

Com os insumos em mãos, o trabalho completo pode ser feito em 4 semanas.

Semana 1

Auditoria & Primitivos

  • Levantar valores do Figma
  • Criar escala de primitivos
  • Validar com design
Semana 2

Tokens Semânticos

  • Mapear intenções de uso
  • Criar semantic.json
  • Aplicar no Figma Variables
Semana 3

Tokens de Componente

  • Um arquivo por componente
  • Ligar ao Figma
  • Review com dev lead
Semana 4

Storybook & Entrega

  • Exportar CSS + JS tokens
  • Documentar no Storybook
  • Handoff para desenvolvimento

Próximo passo imediato: compartilhar acesso ao Figma e paleta de cores para iniciar a auditoria.

// gerado pelo Style Dictionary · não editar
import UIKit
import SwiftUI

public enum DSTokens {

  public enum Color {
    public enum Brand {
      public static let primary = UIColor(
        red: 0, green: 0.278, blue: 0.8, alpha: 1)
      public static let dark = UIColor(
        red: 0, green: 0.102, blue: 0.361, alpha: 1)
    }
    public enum Feedback {
      public static let error   = UIColor(hex: "#CC2200")
      public static let success = UIColor(hex: "#0A7A40")
    }
  }

  public enum Spacing {
    public static let insetSm: CGFloat = 8
    public static let insetMd: CGFloat = 16
    public static let insetLg: CGFloat = 24
    public static let stackMd: CGFloat = 16
  }

  public enum Radius {
    public static let button: CGFloat = 8
    public static let card:   CGFloat = 16
  }
}
uso em SwiftUI

struct PSButton: View {

      .foregroundColor(.white)

  }
}

// DSTokens.kt
object DSTokens {
  val ColorBrandPrimary = Color(0xFF0047CC)
  val ColorBrandDark    = Color(0xFF001A5C)
  val ColorFeedbackError = Color(0xFFCC2200)
  val SpacingInsetSm = 8.dp
  val SpacingInsetMd = 16.dp
  val RadiusButton = 8.dp
}

// PSButton.kt
@Composable
fun PSButton(label: String, onClick: () -> Unit) {
  Button(
    onClick = onClick,
    colors = ButtonDefaults.buttonColors(
      containerColor = DSTokens.ColorBrandPrimary),
    shape = RoundedCornerShape(DSTokens.RadiusButton),
    contentPadding = PaddingValues(
      horizontal = DSTokens.SpacingInsetMd,
      vertical   = DSTokens.SpacingInsetSm)
  ) { Text(label) }
}

// token.service.ts — tema em runtime
@Injectable({ providedIn: 'root' })
export class TokenService {
  setTheme(theme: 'porto' | 'dark') {
    const root = document.documentElement
    const map = THEMES[theme]
    Object.entries(map).forEach(([k, v]) =>
      root.style.setProperty(k, v as string))
  }
}
