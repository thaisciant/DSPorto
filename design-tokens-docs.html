Arquitetura de Coleções

A estrutura segue uma hierarquia estrita de 5 coleções. Um token nunca referencia uma coleção de nível não-imediatamente-anterior — isso garante rastreabilidade e controle total de mudanças.

01
Primitive
Valores absolutos e brutos. Nenhuma semântica.
→
02
Brand
Identidade da marca. Referencia Primitive.
→
03
Theme
Variações de contexto (light / dark). Referencia Brand.
→
04
Channel
Customização por produto / canal. Referencia Theme.
→
05
Platform
Saída técnica por plataforma. Consome Channel.
Regra fundamental: nenhum componente ou coleção consome valores de Primitive diretamente em produção. O consumo direto de primitivos é exclusivo da coleção Brand.

Grupos presentes em todas as coleções

● color ● typography ● numeric-values ● breakpoint ● aspect-ratio

Coleção 05

Platform

A coleção Platform não contém novos tokens — ela é o processo de transformação. O Style Dictionary lê os JSONs das coleções anteriores e gera os artefatos nativos de cada plataforma.

// token.service.ts — troca de tema em runtime
import { Injectable } from '@angular/core'

type ThemeName = 'light' | 'dark'

const THEMES: Record<ThemeName, Record<string, string>> = {
  light: {
    '--theme-color-surface-default': '#FFFFFF',
    '--theme-color-text-primary':    '#1A1A2E',
    '--theme-color-interactive':     '#0047CC',
    '--theme-color-border-default':  '#D0D0E8',
  },
  dark: {
    '--theme-color-surface-default': '#1A1A2E',
    '--theme-color-text-primary':    '#F4F5F9',
    '--theme-color-interactive':     '#5B9BFF',
    '--theme-color-border-default':  '#3D3D5C',
  },
}

@Injectable({ providedIn: 'root' })
export class TokenService {
  private current: ThemeName = 'light'

  setTheme(theme: ThemeName): void {
    const root = document.documentElement
    Object.entries(THEMES[theme]).forEach(([k, v]) =>
      root.style.setProperty(k, v))
    this.current = theme
  }

  getTheme(): ThemeName { return this.current }
}
