# Contract: Button Android Component API (Kotlin / Jetpack Compose)

**Platform**: Android API 26+ — Jetpack Compose 1.6+ + Kotlin 2.x
**Module**: `:button-android`
**Date**: 2026-06-15

---

## Public API

### Composable signature

```kotlin
@Composable
fun PSButton(
    label: String,                         // required; non-empty
    onPress: () -> Unit,                   // required
    modifier: Modifier = Modifier,
    variant: ButtonVariant = ButtonVariant.Primary,
    size: ButtonSize = ButtonSize.MD,
    isDisabled: Boolean = false,
    isLoading: Boolean = false,
    fullWidth: Boolean = false,
    iconLeading: ImageVector? = null,
    iconTrailing: ImageVector? = null,
    accessibilityLabel: String? = null,    // null → uses label
)
```

### MVI — Intent and UiState

```kotlin
sealed class ButtonIntent {
    object Press          : ButtonIntent()
    object HoverEnter     : ButtonIntent()  // mouse/stylus on large-screen
    object HoverExit      : ButtonIntent()
    data class Configure(
        val variant: ButtonVariant,
        val size: ButtonSize,
        val isDisabled: Boolean,
        val isLoading: Boolean,
    ) : ButtonIntent()
}

data class ButtonUiState(
    val variant: ButtonVariant  = ButtonVariant.Primary,
    val size: ButtonSize        = ButtonSize.MD,
    val interactionState: ButtonInteractionState = ButtonInteractionState.Default,
    val isDisabled: Boolean     = false,
    val isLoading: Boolean      = false,
    val tokenSet: ButtonTokenSet = ButtonTokenSet.default(),
)

enum class ButtonInteractionState { Default, Hover, Focus, Active, Disabled, Loading }
```

### ViewModel

```kotlin
class PSButtonViewModel : ViewModel() {
    private val _uiState = MutableStateFlow(ButtonUiState())
    val uiState: StateFlow<ButtonUiState> = _uiState.asStateFlow()

    fun processIntent(intent: ButtonIntent)
    fun resolveTokenSet(variant: ButtonVariant, state: ButtonInteractionState): ButtonTokenSet
}
```

## Token object consumed

All values from `PSButtonTokens.kt` — generated by Style Dictionary. No Color
literals or hard-coded Dp/sp values in `PSButton.kt` or `PSButtonViewModel.kt`.

```kotlin

    object Color {
        object Background {
            val Primary         = Color(0xFF0047CC)
            val PrimaryHover    = Color(0xFF003087)
```

## Accessibility contract

| Modifier | Value | When |
|----------|-------|------|
| `semantics { role = Role.Button }` | — | Always |
| `semantics { contentDescription = accessibilityLabel ?: label }` | — | Always |
| `semantics { disabled() }` | — | When `isDisabled \|\| isLoading` |
| `semantics { stateDescription = "Loading" }` | — | When `isLoading = true` |
| `.clickable(enabled = !isDisabled && !isLoading)` | — | Always |
| Focus ring | 2 dp border in `PSButtonTokens.Color.Border.FocusRing` via custom `Indication` | On focused state |
| Minimum touch target | `Modifier.defaultMinSize(minWidth = 44.dp, minHeight = 44.dp)` | Always |

## Invariants (test-verifiable)

1. `label` MUST NOT be empty — `require(label.isNotBlank())` at ViewModel init.
2. Pressing a disabled or loading Button MUST NOT call `onPress`.
3. All colour values MUST originate from `PSButtonTokens` — no `Color(0x...)` literals
   in `PSButton.kt` or `PSButtonViewModel.kt`.
4. `ButtonIntent.Configure(isLoading = true)` sets `isDisabled = true` in UiState.
5. `PSButtonViewModel.resolveTokenSet` MUST return the correct token set for each
   of the 72 (4 variant × 6 state) permutations — covered by unit tests.
