# Contract: Button Web Component API (React / TypeScript)

**Platform**: Web — React 18 + TypeScript 5
**Package**: `@ps/ds-essential/button-web`
**Date**: 2026-06-15

---

## Public API

### Component signature

```typescript
import { Button } from '@ps/ds-essential/button-web'

<Button
  label="Publicar bundle"
  variant="primary"        // ButtonVariant — optional, default "primary"
  size="md"                // ButtonSize — optional, default "md"
  onPress={() => {}}       // () => void — required
  isDisabled={false}       // boolean — optional, default false
  isLoading={false}        // boolean — optional, default false
  fullWidth={false}        // boolean — optional, default false
  iconLeading={<TbPlus />} // ReactNode — optional
  iconTrailing={null}      // ReactNode — optional
  accessibilityLabel=""    // string — optional, defaults to label
/>
```

### Types

```typescript
type ButtonVariant = 'primary' | 'secondary' | 'ghost' | 'destructive'
type ButtonSize    = 'sm' | 'md' | 'lg'

interface ButtonProps {
  label:              string           // required; non-empty
  variant?:           ButtonVariant    // default: 'primary'
  size?:              ButtonSize       // default: 'md'
  onPress:            () => void       // required
  isDisabled?:        boolean          // default: false
  isLoading?:         boolean          // default: false; implies isDisabled
  fullWidth?:         boolean          // default: false
  iconLeading?:       React.ReactNode  // optional
  iconTrailing?:      React.ReactNode  // optional
  accessibilityLabel?: string          // default: label value
}
```

## CSS Custom Properties consumed

All values come from `channel.component.button.*` tokens via CSS variables
generated by Style Dictionary. The component MUST NOT contain any literal
hex, px, or font-weight values outside of token variable references.

```css
/* Colour — Primary (representative; full list in button-tokens.html) */
--btn-color-bg-primary
--btn-color-bg-primary-hover
--btn-color-bg-primary-active
--btn-color-bg-primary-disabled
--btn-color-fg-primary
--btn-color-fg-primary-disabled
--btn-color-focus-ring

/* Colour — Secondary */
--btn-color-bg-secondary-hover
--btn-color-fg-secondary
--btn-color-border-secondary
--btn-color-border-secondary-disabled

/* Shape */
--btn-radius
--btn-border-width-secondary
--btn-focus-ring-width
--btn-focus-ring-offset

/* Size — MD (representative) */
--btn-height-md
--btn-padding-x-md
--btn-padding-y-md
--btn-icon-size-md
--btn-gap-md
--btn-font-size-md
--btn-font-weight
```

## Rendered HTML structure

```html
<button
  class="btn btn--{variant} btn--{size} [btn--full-width] [btn--loading] [btn--disabled]"
  disabled?           <!-- present only when isDisabled or isLoading -->
  aria-disabled?      <!-- present when isDisabled, so screen readers announce it -->
  aria-label="{accessibilityLabel}"
  aria-busy="{isLoading}"
>
  <!-- icon-leading slot (optional) -->
  <span class="btn__icon btn__icon--leading" aria-hidden="true">
    {iconLeading}
  </span>

  <!-- label slot (always) -->
  <span class="btn__label">{label}</span>

  <!-- icon-trailing slot (optional) -->
  <span class="btn__icon btn__icon--trailing" aria-hidden="true">
    {iconTrailing}
  </span>

  <!-- loading spinner (replaces icon-leading when isLoading) -->
  <span class="btn__spinner" aria-hidden="true" role="presentation">
    <!-- CSS-animated spinner -->
  </span>
</button>
```

## Responsive behaviour

At viewport widths < `channel.breakpoint.tablet` (768 px), buttons with
`fullWidth={true}` render at 100% container width. This is a CSS-only
implementation via a media query in the component stylesheet:

```css
@media (max-width: 767px) {
  .btn--full-width { width: 100%; }
}
```

The button never self-detects viewport width at runtime.

## Accessibility contract

| Attribute | Value | When |
|-----------|-------|------|
| `role` | `button` (implicit on `<button>`) | Always |
| `aria-label` | `accessibilityLabel` prop | When provided; defaults to `label` |
| `aria-disabled` | `"true"` | When `isDisabled = true` |
| `aria-busy` | `"true"` | When `isLoading = true` |
| `disabled` (HTML) | Present | When `isDisabled` or `isLoading` |
| Focus ring | 2px `outline` in `--btn-color-focus-ring`, offset 2px | On `:focus-visible` |

---

## Invariants (test-verifiable)

1. `label` MUST NOT be empty — throw in development, no-op in production.
2. Clicking a disabled or loading Button MUST NOT call `onPress`.
3. All colour values MUST resolve to a CSS variable — no literal hex in rendered styles.
4. The focus ring MUST be visible when the button has keyboard focus.
5. Contrast of foreground on background MUST be ≥ 4.5:1 for all non-disabled states.
